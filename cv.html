I have used different development environments (e.g. Microsoft Visual Studio, Eclipse, RStudio, and vim/emacs with different plugins) for extended periods of time; for the last years I have been using mostly the RStudio and Emacs environments.

I have developed using many different languages. From MIX (the language Knuth uses in The art of computer programming) in which I implemented Thompson NFA for regular expressions. html, css, php, and javascript for website development. C++, Python and java for learning these languages a bit and playing with different technologies. C++ with SQL and web languages while I was at CoachR. To mostly Clojure and R at Adgoji.

For my current work R is the most suited, supported by clojure and java for interacting with the hadoop stack. To extract additional speed and interoperability I am looking into Rcpp to get C++ speeds and use of C++ libraries while working in the interactive R environment.
